## Component Design

### 1. Core Risk Engine (`crates/engine`)

Pure business logic library crate. No network calls, no blockchain dependency.

**Key types:**

- `RiskSpectrum` вЂ” enum: `Sovereign`, `StablecoinSavings` (extensible for future positions)
- `VaultConfig` вЂ” approved yield sources, exposure caps per source, concentration limits, auto-derisking thresholds
- `Portfolio` вЂ” current allocations across adapters, deposit/withdrawal/yield tracking
- `RiskAssessment` вЂ” health score, breach flags, recommended actions

**Core operations:**

- `evaluate_risk(portfolio, market_data) -> RiskAssessment`
- `recommend_allocation(config, deposit_amount) -> AllocationPlan`
- `should_derisk(portfolio, health_checks) -> DeriskAction`

The engine takes data in, returns decisions. I/O is handled by API and Sentinel layers.

### 2. Adapter Layer (`crates/adapters`)

Trait-based plugin system:

```rust
#[async_trait]
pub trait YieldAdapter: Send + Sync {
    fn name(&self) -> &str;
    fn risk_position(&self) -> RiskSpectrum;
    async fn deposit(&self, amount: U256) -> Result<TxRequest>;
    async fn withdraw(&self, amount: U256) -> Result<TxRequest>;
    async fn current_yield(&self) -> Result<f64>;
    async fn health_check(&self) -> Result<HealthStatus>;
    async fn tvl(&self) -> Result<U256>;
}
```

**M1-3 implementations:**

- `SovereignBondAdapter` вЂ” wraps MockRWAVault on testnet (simulates tokenised sovereign bonds at ~4-5% APY). Interface identical to what a real RWA provider adapter would use вЂ” swap the contract address for production.
- `AaveSavingsAdapter` вЂ” wraps Aave V3 on Sepolia. Deposits USDC/USDT into lending pools. Earns real testnet supply rates.

**Adapter template** (`adapter-template/`): scaffold with trait implementation, example tests, Cargo.toml, and integration guide.

### 3. Smart Contracts (`contracts/`)

**ImpactVault.sol** вЂ” ERC-4626 vault with:
- Whitelisted depositors (gating for verified organisations)
- Configurable yield source routing
- Timelock on parameter changes
- Emergency derisk function (can only reduce risk exposure)

**YieldSplitter.sol** вЂ” yield distribution:
- Array of `(address, basisPoints)` recipient pairs
- On-chain event for every disbursement
- Admin updates recipients via timelock

**MockRWAVault.sol** вЂ” testnet-only:
- ERC-4626 interface
- Mints synthetic yield at configurable APY
- Simulates sovereign bond behaviour for full pipeline testing

### 4. Sentinel Bot (`crates/sentinel`)

Long-running Tokio async binary:

- Polls adapter health checks on configurable interval (default 60s)
- Monitors: smart contract risk indicators, oracle freshness, liquidity depth, utilisation rates, collateral ratios
- Evaluates thresholds from `VaultConfig`
- On threshold breach: calls `should_derisk()` -> submits migration tx if needed
- Structured JSON logging to stdout, optional webhook notifications

### 5. API Server (`crates/api`)

Axum REST API (testnet, no auth in M1-3):

- `GET /vault/{id}/status` вЂ” portfolio, allocations, health
- `GET /vault/{id}/risk` вЂ” risk assessment
- `POST /vault/{id}/deposit` вЂ” generate unsigned deposit tx
- `POST /vault/{id}/withdraw` вЂ” generate unsigned withdraw tx
- `GET /adapters` вЂ” list adapters and health
- `GET /adapters/{name}/health` вЂ” detailed health check
- `GET /yield/history` вЂ” yield generation history
- `GET /disbursements` вЂ” on-chain disbursement log

Returns unsigned transaction payloads. Caller signs.
